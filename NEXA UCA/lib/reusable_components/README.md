# Reusable Components 

Esta carpeta contiene todos los componentes reutilizables utilizados dentro de la interfaz de usuario
NEXA UCA 

El objetivo principal de esta sección es centralizar elementos visuales compartidos para mantener 
consistencia en el diseño,reutilizar código y facilitar el mantenimiento de la aplicación 

Los componentes almacenados aqui pueden ser utilizados en múltiples pantallas sin necesidad de recrear
su estructura visual o funcional 

---

## Responsabilidades

- Reutilización de componentes UI
- Consistencia visual entre pantallas
- Reducción de código duplicado 
- Centralizadción de elementos gráficos y funcionales
- Facilitar mantenimiento y escalabilidad 

----
## Estructura 

### buttons/

Contiene botones reutilizables utilizados dentro de la aplicacion 

Ejemplos:
- Botón principal
- Botón secundario
- Botón de confirmación 
- Botón de cancelar 
- Botón de cerrar sesión 

---

### cards/

Contiene tarjetas visuales reutilizables utilizadas en dashboards y módulos de información

Ejemplos:

- Cards de productividad 
- Cards de actividades
- Cards de estadisticas 
- Cards de solicitudes 
- Cards informativas 

---
### modals/

Contiene tarjetas visuales reutilizables utilizadas en dashboards y módulos de información 

Ejemplos:

- Validación GPS 
- Confirmación de registro
- Solicitud enviada 
- Acceso denegado
- Mensajes de estado 

----

### badges/

Contiene componentes visuales utilizados para representar estados dentro de la aplicación 

Ejemplos:

- Pendiente
- Aprobado
- Rechazado
- En Progreso

---

### inputs/
Contiene campos de entrada reutilizables.

Ejemplos:

- Textfield personalizado
- PasswordField
- SearchBar
- TextArea
- Input Bloqueado

---
### navigation/

Contiene componentes relaciones con navegación y estructura visual general

Ejemplos: 

- Headers
- Botton navigation 
- Navbar 
- Wave Footer
- Secciones superiores

---

### charts/

Contiene componentes gráficos reutilizables utilizados para visualización de progreso y estadísticas 

Ejemplos: 

- Progress Circle
- Pie Charts
- Progress Bars 
- Indicadores de horas acumuladas

---

# Objetivo arquitectónico 

La separación de componentes reutilizables permite mantener una arquitectura más limpia , modular y
escalable, facilitando la construcción de nuevas pantallas sin necesidad de duplicar código visual o funcional
